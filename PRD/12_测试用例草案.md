## 12.4 E2E测试用例

### E2E-01：完整的"一句话报销"场景

**前置条件**：
- 系统已启动
- 已有published的差旅报销流程模板
- 测试用户已创建

**步骤**：

```
1. 打开登录页，输入用户名密码，点击登录
   → 跳转到对话工作台

2. 在对话框输入"我要报销去北京的差旅费1200元"
   → 系统识别意图为create_submission
   → 系统匹配差旅报销流程
   → 系统追问"请问出差日期是？"

3. 输入"3月1日到3月3日"
   → 系统生成草稿
   → 展示草稿预览（金额=1200，目的地=北京，日期=3/1-3/3）

4. 点击"确认提交"
   → 系统提交到OA
   → 返回"提交成功，申请编号#12345678"

5. 点击"我的申请"
   → 看到刚提交的申请
   → 状态为"已提交"

6. 点击申请详情
   → 看到状态时间线
```

**验收标准**：
- 全流程 ≤ 2分钟
- 对话轮次 ≤ 3轮
- 提交成功

### E2E-02：Bootstrap初始化场景

**前置条件**：
- 系统已启动
- 管理员账户已创建
- OpenAPI型OA fixture可用

**步骤**：

```
1. 管理员登录
2. 进入初始化中心
3. 点击"创建初始化任务"
4. 填写OA地址和OpenAPI文档地址
5. 点击"创建"
   → 任务创建成功，状态=CREATED

6. 等待自动处理
   → DISCOVERING → PARSING → NORMALIZING → COMPILING → REPLAYING → REVIEW

7. 查看OCL报告
   → 包含coverage/confidence/risk/evidence/recommendation

8. 点击"发布"
   → 状态=PUBLISHED
   → 流程库出现新的流程模板
```

**验收标准**：
- 全流程 ≤ 15分钟
- OCL报告完整
- 流程模板成功发布

---

### E2E-03：权限拒绝+审计场景

**前置条件**：
- 系统已启动
- 普通教师账户（无大额采购权限）

**步骤**：

```
1. 教师登录
2. 输入"我要采购一台服务器，预算50万"
   → 系统识别意图
   → 权限校验失败
   → 返回拒绝原因："您当前角色没有大额采购权限"
   → 提供替代建议

3. 审计员登录
4. 进入审计日志页面
5. 查询该教师的操作记录
   → 看到权限拒绝的审计日志
   → 包含完整的决策过程
```

**验收标准**：
- 拒绝原因清晰
- 审计日志完整
- Trace可追溯
