| 客户端→API | HTTPS/TLS 1.2+ | ⏳ HTTP | ✅ 必须 |

| API→Redis | TLS | ⏳ 明文 | ✅ 推荐 |

### 11.2.2 数据安全

| 数据类型 | 存储方式 | 访问控制 |
|----------|----------|----------|
| 用户密码 | bcrypt哈希 | 不可逆 |
| OA认证凭据 | AES-256加密 | 仅系统访问 |
| 表单数据 | 明文（MVP）/ 字段级加密（生产） | 租户隔离 |
| 审计日志 | append-only | 只读（审计员） |
| 会话Token | JWT签名 | 有效期控制 |

| 要求 | 措施 | 状态 |

## 11.6 漏洞管理

### 漏洞响应流程

```
发现漏洞
  │
  ▼
评估严重性
  ├── Critical（P0）：2小时内响应，24小时内修复
  ├── High（P1）：24小时内响应，72小时内修复
  ├── Medium（P2）：72小时内响应，2周内修复
  └── Low（P3）：1周内响应，下个版本修复
  │
  ▼
修复 → 测试 → 发布 → 通知
```

## 11.7 隐私设计

### 隐私原则

1. **数据最小化**：仅收集业务必需的最少数据
2. **目的限制**：数据仅用于声明的目的
3. **存储限制**：超过保留期的数据自动清理
4. **透明性**：用户可查看自己的数据使用情况
5. **安全性**：适当的技术和组织措施保护数据

### 数据分类

| 分类 | 示例 | 保护级别 |
|------|------|----------|
| 公开数据 | 流程模板名称、分类 | 低 |
| 内部数据 | 用户名、部门、角色 | 中 |
| 敏感数据 | 表单内容、金额、附件 | 高 |
| 机密数据 | 密码、认证凭据、密钥 | 极高 |

### 数据保留策略

| 数据类型 | 保留期 | 到期处理 |
|----------|--------|----------|
| 审计日志 | ≥ 365天 | 归档到冷存储 |
| 业务日志 | ≥ 180天 | 删除 |
| 对话记录 | 90天 | 匿名化 |
| 用户数据 | 账户存续期 | 账户注销后30天删除 |
| 临时文件 | 24小时 | 自动清理 |
