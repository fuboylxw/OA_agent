## 4.1 Bootstrap 初始化状态机

### 状态流转

```
CREATED → DISCOVERING → PARSING → NORMALIZING → COMPILING → REPLAYING → REVIEW → PUBLISHED
   │           │            │           │            │           │          │
   └───────────┴────────────┴───────────┴────────────┴───────────┘          │
                              任意阶段失败 → FAILED                         │
                                              │                             │
                                              └── 重试 → CREATED ──────────┘
```

### 状态定义

| 状态 | 含义 | 触发条件 | 允许操作 |
|------|------|----------|----------|
| CREATED | 任务已创建 | 用户提交初始化请求 | 取消 |
| DISCOVERING | 识别中 | 系统开始自动识别OA | 无 |
| PARSING | 解析中 | 识别完成，开始解析流程/字段 | 无 |
| NORMALIZING | 归一化 | 解析完成，生成IR中间表示 | 无 |
| COMPILING | 编译中 | IR生成完成，编译适配器 | 无 |
| REPLAYING | 回放测试 | 编译完成，执行回放验证 | 无 |
| REVIEW | 待审核 | 回放通过，等待管理员审核 | 发布、驳回 |
| PUBLISHED | 已发布 | 管理员确认发布 | 下线 |
| FAILED | 失败 | 任意阶段出错 | 重试、删除 |

### 状态转换规则

```
允许的转换：
  CREATED      → DISCOVERING
  DISCOVERING  → PARSING | FAILED
  PARSING      → NORMALIZING | FAILED
  NORMALIZING  → COMPILING | FAILED
  COMPILING    → REPLAYING | FAILED
  REPLAYING    → REVIEW | FAILED
  REVIEW       → PUBLISHED | FAILED
  FAILED       → CREATED（重试）

不允许的转换：
  PUBLISHED    → 任何状态（终态，需新建任务）
  任何状态     → 跳跃转换（必须按顺序）
```

## 4.2 对话编排状态机

### 主链路状态流转

```
RECEIVED → INTENT_PARSED → FLOW_MATCHED → PERMISSION_CHECKED
                                              │
                              ┌────────────────┤
                              │ allowed        │ denied
                              ▼                ▼
                         FORM_DRAFTED     DENIED（返回拒绝原因）
                              │
                              ▼
                        RULE_VALIDATED
                         │         │
                    pass │         │ fail
                         ▼         ▼
                     SUBMITTED   VALIDATION_ERROR（返回错误）
                         │
                         ▼
                    STATUS_SYNCED
                         │
                         ▼
                      COMPLETED
```

### 降级路径

```
任意阶段异常 → DEGRADED
  │
  ├── LLM不可用 → 返回"服务暂时不可用"
  ├── OA不可达 → 返回"OA系统暂时不可用，已保存草稿"
  └── 超时 → 返回"处理超时，请稍后重试"

DEGRADED → FAILED（多次重试失败）
```

### 状态定义

| 状态 | 含义 | Agent | 超时 |
|------|------|-------|------|
| RECEIVED | 收到用户消息 | - | - |
| INTENT_PARSED | 意图已识别 | intent_agent | 3s |
| FLOW_MATCHED | 流程已匹配 | flow_agent | 2s |
| PERMISSION_CHECKED | 权限已校验 | auth_agent | 2s |
| FORM_DRAFTED | 草稿已生成 | form_agent | 5s |
| RULE_VALIDATED | 规则已校验 | rule_agent | 2s |
| SUBMITTED | 已提交OA | submit_agent | 10s |
| STATUS_SYNCED | 状态已同步 | status_agent | 5s |
| COMPLETED | 流程完成 | audit_agent | 2s |
| DENIED | 权限拒绝 | auth_agent | - |
| DEGRADED | 降级处理 | fallback_agent | - |
| FAILED | 失败 | - | - |

## 4.3 提交（Submission）状态机

### 状态流转

```
DRAFT → VALIDATING → SUBMITTING → SUBMITTED → SYNCING
                                      │
                          ┌───────────┼───────────┐
                          ▼           ▼           ▼
                      APPROVED    REJECTED    IN_PROGRESS
                                      │
                                      ▼
                                  SUPPLEMENTING → RESUBMITTED
                                                      │
                                                      ▼
                                                  SUBMITTED（回到同步）

用户操作：
  SUBMITTED/IN_PROGRESS → CANCELLED（撤回）
  SUBMITTED/IN_PROGRESS → URGED（催办，不改变主状态）
  REJECTED → SUPPLEMENTING（补件）
  SUBMITTED/IN_PROGRESS → DELEGATED（转办）
```

### 状态定义

| 状态 | 含义 | 触发条件 | 允许操作 |
|------|------|----------|----------|
| DRAFT | 草稿 | 对话生成草稿 | 编辑、提交、删除 |
| VALIDATING | 校验中 | 用户确认提交 | 无 |
| SUBMITTING | 提交中 | 规则校验通过 | 无 |
| SUBMITTED | 已提交 | OA接收成功 | 撤回、催办、转办 |
| IN_PROGRESS | 审批中 | OA开始审批 | 撤回、催办、转办 |
| APPROVED | 已通过 | OA审批通过 | 无（终态） |
| REJECTED | 已驳回 | OA审批驳回 | 补件 |
| CANCELLED | 已撤回 | 用户撤回 | 无（终态） |
| SUPPLEMENTING | 补件中 | 用户补充材料 | 提交 |
| RESUBMITTED | 已重提 | 补件后重新提交 | 同SUBMITTED |
| FAILED | 失败 | 提交OA失败 | 重试 |

### 幂等提交规则

```
提交请求携带 idempotency_key（UUID）
  │
  ├── 首次提交 → 正常处理，记录 idempotency_key
  │
  └── 重复提交（相同 idempotency_key）
       │
       ├── 前次成功 → 返回前次结果（不重复执行）
       └── 前次失败 → 允许重试
```

### 办事动作与状态约束

| 动作 | 意图代号 | 前置状态 | 权限要求 | 二次确认 | OA接口 |
|------|----------|----------|----------|----------|--------|
| 撤回 | cancel_submission | SUBMITTED/IN_PROGRESS | 本人 | ✅ 是 | cancel |
| 催办 | urge | SUBMITTED/IN_PROGRESS | 本人 | ✅ 是 | urge |
| 补件 | supplement | REJECTED | 本人 | ❌ 否 | supplement |

## 4.5 队列与异步流程

### 队列定义

| 队列名 | 用途 | 并发数 | 重试次数 | 超时 |
|--------|------|--------|----------|------|
| bootstrap.queue | Bootstrap流水线 | 3 | 3 | 15min |
| parse.queue | 流程解析 | 5 | 3 | 5min |
| submit.queue | 提交到OA | 10 | 3 | 30s |
| status.queue | 状态同步 | 10 | 5 | 10s |
| audit.queue | 审计日志写入 | 20 | 3 | 5s |

### 重试策略

```
重试间隔：指数退避
  第1次重试：5秒后
  第2次重试：30秒后
  第3次重试：5分钟后

死信队列：
  超过最大重试次数 → 进入死信队列
  死信队列消息 → 人工处理 + 告警通知
```

### 补偿机制

```
提交失败补偿流程：
  1. 记录失败原因
  2. 标记submission状态为FAILED
  3. 发送告警通知
  4. 用户可手动重试

状态同步失败补偿：
  1. 使用上次已知状态
  2. 定时轮询重试（每5分钟）
  3. 超过24小时未同步 → 告警
```

## 4.6 关键业务流程

### 流程1：完整的"一句话办事"流程

```
用户输入 "我要报销差旅费1200元"
  │
  ▼
[intent_agent] 识别意图 → create_submission
  │
  ▼
[flow_agent] 匹配流程 → EXPENSE_CLAIM（差旅报销）
  │
  ▼
[auth_agent] 权限校验
  ├── 平台权限：用户角色=teacher，有报销权限 ✅
  └── OA权限：调用OA接口确认 ✅
  │
  ▼
[form_agent] 字段提取
  ├── 已提取：金额=1200，类型=差旅
  └── 缺失：出差日期、目的地 → 追问用户
  │
  ▼
用户补充 "3月1日到3日去北京"
  │
  ▼
[form_agent] 补充字段 → 生成草稿
  │
  ▼
展示草稿给用户确认
  │
  ▼
用户确认提交
  │
  ▼
[rule_agent] 规则校验
  ├── 金额≤5000 ✅
  ├── 日期合法 ✅
  └── 必填字段完整 ✅
  │
  ▼
[submit_agent] 幂等提交到OA
  │
  ▼
[status_agent] 获取初始状态
  │
  ▼
[audit_agent] 记录审计日志
  │
  ▼
返回用户：提交成功，申请编号 #12345678
```

### 流程2：Bootstrap全链路

```
管理员创建任务（提供OA地址）
  │
  ▼
[DISCOVERING] oa_discovery_agent
  ├── 识别OA厂商/版本
  ├── 识别认证方式
  ├── 识别接口特征
  └── 产出OCL等级初评
  │
  ▼
[PARSING] schema_parser_agent + mapping_agent
  ├── 抽取流程列表
  ├── 抽取字段定义
  ├── 抽取规则约束
  └── 映射到Canonical Schema
  │
  ▼
[NORMALIZING] IR生成
  ├── FlowIR（流程中间表示）
  ├── FieldIR（字段中间表示）
  ├── RuleIR（规则中间表示）
  └── PermissionIR（权限中间表示）
  │
  ▼
[COMPILING] adapter_compiler
  ├── 从IR生成连接器代码
  └── API优先，RPA降级
  │
  ▼
[REPLAYING] replay_validator
  ├── 执行回放测试用例
  └── 验证适配器可用性
  │
  ▼
[REVIEW] 管理员审核
  ├── 查看OCL报告
  ├── 确认流程模板
  └── 点击发布
  │
  ▼
[PUBLISHED] 流程模板发布到流程库
```

## 4.7 异常处理流程

### 异常分类

| 异常类型 | 示例 | 处理方式 |
|----------|------|----------|
| 用户输入异常 | 意图无法识别 | 引导用户重新描述 |
| 权限异常 | 无权限 | 返回拒绝原因+替代路径 |
| 规则异常 | 字段校验失败 | 返回具体错误+修改建议 |
| OA异常 | OA系统不可达 | 保存草稿+稍后重试 |
| 系统异常 | 内部错误 | 记录日志+返回友好提示 |
| 超时异常 | 处理超时 | 异步处理+轮询结果 |

### 降级策略

```
LLM不可用：
  → 使用规则引擎进行简单意图匹配
  → 返回"智能助手暂时不可用，请使用快捷操作"

OA不可达：
  → 保存草稿到本地
  → 加入重试队列
  → 通知用户"已保存草稿，将在OA恢复后自动提交"

数据库不可用：
  → 返回503错误
  → 触发告警
```
